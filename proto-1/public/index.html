How the Flow Works

  1. Client sends HTTP request to proxy on :3000
  2. Proxy matches the URL path against config.json routes
  3. If route has auth: true, JWT is verified against NeonDB sessions
  4. Proxy strips the prefix (if configured) and forwards to the hidden backend port
  5. Backend processes the request and responds
  6. Proxy pipes the response back to the original client
  7. All 4 hops are streamed live to this UI via SSE
