Sliding Window Log (SWL)

  1. Every allowed request appends a timestamp to a per-(IP, endpoint) log.
  2. On each incoming request, timestamps older than windowMs are evicted from the front of the log.
  3. If log.length ≥ limit → reject 429 immediately, before any proxy forwarding.
  4. Otherwise allow: push current timestamp and continue.

Why SWL over Token Bucket?

  1. Exact — no approximation error; every request in the window is counted precisely.
  2. Burst-safe — a burst at t=59s doesn't reset at t=60s; the window slides continuously.
  3. Brute-force proof — perfect fit for the POST /login 5 req/min rule.

Trade-off

Memory: O(limit) timestamps per active key. For 100 req/min rules at high traffic, consider Redis for multi-node deployments.
